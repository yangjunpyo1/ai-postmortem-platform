## 2. 구성 요소

### 외부 서비스
| 서비스 | 역할 |
|---|---|
| User | 웹 대시보드 접근 및 슬랙 장애 대응 |
| Slack | 장애 알림 수신 및 대응 대화 |
| Claude API | 슬랙 대화 + CloudWatch 로그 분석 및 Postmortem 문서 생성 |

### AWS 글로벌 서비스
| 서비스 | 역할 |
|---|---|
| CloudFront | HTTPS 적용 및 프론트엔드 CDN |

### AWS 리전 서비스 (ap-northeast-2)
| 서비스 | 역할 |
|---|---|
| S3 | React 프론트엔드 정적 파일 호스팅 |
| CloudWatch | 서버 메트릭 모니터링 및 장애 감지 |
| SNS | 장애 알람 메시지 발행 |
| SQS | 메시지 큐 보관 (유실 방지) |
| API Gateway | 외부 요청을 Lambda로 전달하는 엔드포인트 |

### VPC 내부 서비스
| 서비스 | 역할 |
|---|---|
| IGW | 외부 인터넷 ↔ VPC 통신 통로 |
| NAT Gateway | Private Subnet → 외부 인터넷 통신 통로 |
| Lambda (FastAPI) | 웹 대시보드 백엔드 API 처리 |
| Lambda (알림 전송) | SQS 메시지 기반 슬랙 심각도별 알림 전송 |
| Lambda (Postmortem 자동화) | 슬랙 대화 수집, CloudWatch 로그 수집, Claude API 호출, 문서 저장 |
| EC2 (Grafana) | CloudWatch 메트릭 시각화 |
| RDS (MySQL) | 장애 정보 및 Postmortem 문서 저장 |

## 3. 네트워크 구성

VPC (10.0.0.0/16)

Public Subnet
├── AZ-a (10.0.10.0/24) → NAT Gateway
└── AZ-c (10.0.100.0/24)

Private Subnet 1 (App Layer)
├── AZ-a (10.0.20.0/24) → Lambda(FastAPI), Lambda(알림 전송), EC2(Grafana)
└── AZ-c (10.0.120.0/24) → Lambda(Postmortem 자동화)

Private Subnet 2 (DB Layer)
├── AZ-a (10.0.30.0/24) → RDS (Primary)
└── AZ-c (10.0.130.0/24) → RDS (Standby)

## 4. 흐름 설명

### 🔵 흐름 1 - 웹 접근
User → Web Browser → CloudFront → S3(React)
→ API Gateway → Lambda(FastAPI) → RDS

### 🔴 흐름 2 - 장애 감지 및 알림
CloudWatch → SNS → SQS → Lambda(알림 전송)
→ NGW → IGW → Slack

### 🟢 흐름 3 - 장애 대응 및 Postmortem 생성
User → Slack → API Gateway → Lambda(Postmortem 자동화)
→ CloudWatch (로그 수집)
→ NGW → IGW → Claude API
→ RDS (문서 저장)
→ NGW → IGW → Slack (완료 알림)

### 🟣 흐름 4 - 모니터링
CloudWatch → EC2(Grafana)
