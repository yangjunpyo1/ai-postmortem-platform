# 📋 기술 스택

## 1. 기술 스택 개요

| 분류 | 기술 | 선택 이유 |
|---|---|---|
| IaC | Terraform | 인프라 코드화, 재현 가능한 환경 구축 |
| Cloud | AWS | 실무 표준, 다양한 서비스 연동 |
| 모니터링 | CloudWatch | AWS 서비스 기본 제공, 별도 설치 불필요 |
| 알림 | CloudWatch Alarm | 심각도별 알림 설정, Alertmanager 불필요 |
| 시각화 | Grafana | CloudWatch 연동, 직관적인 대시보드 |
| AI | Claude API | 자연어 분석 및 문서 자동 생성 |
| 연동 | Slack API | 장애 대응 채널 연동 |
| 백엔드 | Python (FastAPI) | Lambda 친화적, AI 연동 쉬움 |
| 프론트엔드 | React | 컴포넌트 기반 UI 개발 |
| DB | AWS RDS (MySQL) | 관계형 데이터 저장, 프리티어 지원 |

## 2. AWS 서비스 상세

| 서비스 | 역할 |
|---|---|
| CloudWatch | 서버 메트릭 모니터링 및 장애 감지 |
| CloudWatch Alarm | 심각도별 임계치 설정 및 알림 트리거 |
| SNS | 장애 감지 시 메시지 발행 |
| SQS | 메시지 큐 보관 (유실 방지) |
| Lambda | 슬랙 대화 수집 및 Claude API 호출 |
| RDS | postmortem 문서 및 장애 데이터 저장 |
| API Gateway | 프론트엔드와 백엔드 연결 통로 |
| S3 | 프론트엔드 정적 파일 호스팅 |
| Secrets Manager | API 키, DB 비밀번호 등 시크릿 관리 |

## 3. 기술 선택 이유 상세

### 3-1. Terraform
인프라를 코드로 관리 (IaC)
terraform apply 한 번으로 전체 AWS 환경 자동 구축
누구든 동일한 환경 재현 가능
이전 개별 프로젝트 Ansible 경험 기반으로
자동화의 중요성을 직접 경험 후 선택

### 3-2. AWS Lambda (서버리스)
24시간 서버 운영 불필요
장애 발생 시에만 실행되어 비용 절감
AWS 프리티어 범위 내 운영 가능

### 3-3. CloudWatch + CloudWatch Alarm
CloudWatch로 AWS 서비스 메트릭 자동 수집
CloudWatch Alarm으로 심각도별 임계치 설정
Critical/Warning/Info 단계별 알림 트리거
Alertmanager 없이도 동일한 기능 구현 가능

### 3-4. SNS + SQS 조합
메시지 유실 방지
SNS 단독 사용 시 슬랙 다운되면 알림 유실
SQS 큐에 보관하면 슬랙 복구 후 자동 전송

### 3-5. Claude API
자연어 기반 문서 자동 생성
슬랙 대화를 분석해서 일관된 형식의 postmortem 문서 생성
한국어 지원

### 3-6. Python (FastAPI)
AWS Lambda가 Python 기본 지원
Claude API 연동 쉬움
FastAPI로 빠른 API 개발 가능

### 3-7. Grafana
CloudWatch와 직접 연동 가능
Prometheus 없이도 사용 가능
직관적인 대시보드로 장애 메트릭 시각화
이전 프로젝트에서 사용 경험 보유
